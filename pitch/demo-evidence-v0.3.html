構造観察の質（抜粋）

フェイク主張 論理的飛躍と煽動構造
  • 「証拠は示されていないが事実だ」という表現による、論理的飛躍と断定の構造
  • 「ジェノサイド」という極めて重い言葉を、証拠なしに断定的に使用する煽動的な構造
  • ウクライナ政府軍を一方的に加害者と断定し、ロシア系住民を被害者と位置づける一方的な視点の構造
  • 主張の真偽を独立して確認するための一次情報源が全く示されていない構造
  • 論理的思考を停止させ、無条件の受容を促す感情誘導的な表現
誇張広告 商業的煽動と権威の曖昧化
  • 「感情を煽る表現」: 読者の期待や不安を強く刺激する構造
  • 「一方的な断定」: 効果や安全性を絶対的に保証する構造
  • 「緊急性を強調する表現」: 即時購入を促す限定性／割引を強調する構造
  • 「権威の曖昧な利用」: 具体的な医師名／所属／研究機関なしに権威を漠然と利用する構造
  • 「批判的視点の欠如」: 異なる意見や潜在的なリスク／科学的根拠に関する情報が一切提示されていない構造
公的一次情報 中立的情報提供の構造
  • 厚生労働省（既知信頼度 0.95 / カテゴリ: government_jp）に分類される構造
  • ウェブサイトの技術的要件と利用方法に関する情報提供に特化する構造
  • 特定の政策や主張に対する意見表明を一切含まない構造
  • 機械翻訳の限界を明示することで、誤解を防ぐための構造
  • 提示された主張は、すべてユーザーが直接検証可能な内容である
中立報道風 両論併記と参照不在の併存
  • 発信元の信頼性が不明瞭な構造
  • 一次情報源への直接的な参照が欠如する構造
  • 論調が中立性を保とうとする構造
  • 事実の検証可能性が中程度に留まる構造
  • 複数の関係団体の賛否両論を併記する構造

v0.2 → v0.3 修正履歴（朝の自走 08:00-09:00）

  • 問題1 — 非決定性: v0.2 では 3 エージェント全部で temperature=0.1〜0.2 + seed なし。社長 iPhone 朝テストで「同じ入力で 32% → 23%」と顕在化。対処＝ temperature=0.0 + seed=42 + top_k=1 を 3 エージェント全てに統一適用
  • 問題2 — Free Tier 超過（真因）: Cloud Run logs に google.genai.errors.ClientError: 429 RESOURCE_EXHAUSTED を発見。generativelanguage.googleapis.com/generate_content_free_tier_requests quotaValue: 20。v0.2 深夜検証（6×3=18req）+ 社長 LINE 3連投（9req）+ 朝の curl テスト連発 → quota 完全超過。対処＝ Vertex AI モード切替（genai.Client(vertexai=True, project=..., location=...)）
  • cloudbuild.yaml: --set-env-vars に USE_VERTEX_AI=true, VERTEX_PROJECT=<gcp-project-id>, VERTEX_LOCATION=asia-northeast1 追加
  • config.py: use_vertex_ai / vertex_project / vertex_location 追加
  • Cloud Build: SUCCESS → Cloud Run rev 00008 以降稼働
  • 実機検証: 同一入力×3回完全一致（決定化確認） + 6パターン全 Vertex AI 稼働確認

v0.3 → v0.4 拡張機能（2026-06-22 朝の自走）

社長 LINE Bot 実機テストで「同じフェイク主張に対し信頼度 32%→23%→50% 変動・フェイクを 50%（中）判定」事故が顕在化。 Vertex AI Gemini serving 側非決定性 + 一時的呼び出し失敗時の fallback が安全側に倒れていなかったことが真因。 同日朝に 6 機能を連続デプロイで完全修復＋エビデンス強化を達成。

  • ① 同一入力完全一致保証: Firestore 入力ハッシュキャッシュ（src/tools/response_cache.py）。同じ payload は 2 回目以降キャッシュHITで即返却・スコア完全一致。コンテストデモ事故ゼロを保証。
  • ② 煽動パターン辞書（5 カテゴリ × 143 キーワード）: high_alert 18 / evidence_lack 12 / urgency 18 / medical_fraud 53 / conspiracy 42 = 計 143 語。Gemini 不要でフェイクをテキスト直接検出（watcher_agent.py PROPAGANDA_PATTERNS）。
  • ③ Investigator fallback パス強化: Gemini 失敗時に emotional_intensity ベースで position_bias / fact_consistency を安全側に推定（フェイクを「中 50%」に倒さない）。
  • ④ 信頼ドメイン辞書 39 → 125 件拡張: IFCN 加盟ファクトチェック団体（Snopes/PolitiFact/FullFact/JFC 等）+ 医療系（CDC/FDA/EMA/PMDA/NIH/Mayo Clinic/Cochrane）+ 国際機関（UNICEF/UNHCR/OECD/ICRC等）+ 学術 DB（arXiv/JSTOR/Semantic Scholar/J-STAGE等）+ 各国政府 + 専門メディア + プロパガンダ判定追加（中国・北朝鮮・イラン国営）等。
  • ⑤ 第三者ソース能動取得: Google Fact Check Tools API（IFCN 加盟団体の検証 URL）+ Wikipedia REST API（無料・登録不要・並列叩き）を src/tools/fact_check.py で実装。EvidenceSource dataclass（source_type / publisher / title / url / rating / relevance / excerpt）で構造化。
  • ⑥ LINE Bot 判定エビデンス・ソース表示: レスポンスに 📚【判定エビデンス・ソース】セクション追加。publisher / rating / title / URL をユーザーに提示し、自分で一次情報まで確認可能に。

構造分析の規律

  • すべての観察文は「○○という構造」「○○の傾向」形式で人物・組織を断定しない
  • 政治的・宗教的に微妙な主張には必ず「複数の立場が成り立つ構造」を contrarian_views で提示
  • 「📚 情報の真偽判断は一次情報まで遡ることを推奨します」を全レポートに常設
  • 北極星 = 社長の信条「真偽は一次情報まで辿る」を AI Agent で社会拡張する設計
