構造観察の質（抜粋）

フェイク主張 論理的飛躍と煽動構造
  • 「証拠は示されていないが事実だ」という表現による、論理的飛躍と断定の構造
  • 「ジェノサイド」という極めて重い言葉を、証拠なしに断定的に使用する煽動的な構造
  • ウクライナ政府軍を一方的に加害者と断定し、ロシア系住民を被害者と位置づける一方的な視点の構造
  • 主張の真偽を独立して確認するための一次情報源が全く示されていない構造
  • 論理的思考を停止させ、無条件の受容を促す感情誘導的な表現
誇張広告 商業的煽動と権威の曖昧化
  • 「感情を煽る表現」: 読者の期待や不安を強く刺激する構造
  • 「一方的な断定」: 効果や安全性を絶対的に保証する構造
  • 「緊急性を強調する表現」: 即時購入を促す限定性／割引を強調する構造
  • 「権威の曖昧な利用」: 具体的な医師名／所属／研究機関なしに権威を漠然と利用する構造
  • 「批判的視点の欠如」: 異なる意見や潜在的なリスク／科学的根拠に関する情報が一切提示されていない構造
公的一次情報 中立的情報提供の構造
  • 厚生労働省（既知信頼度 0.95 / カテゴリ: government_jp）に分類される構造
  • ウェブサイトの技術的要件と利用方法に関する情報提供に特化する構造
  • 特定の政策や主張に対する意見表明を一切含まない構造
  • 機械翻訳の限界を明示することで、誤解を防ぐための構造
  • 提示された主張は、すべてユーザーが直接検証可能な内容である
中立報道風 両論併記と参照不在の併存
  • 発信元の信頼性が不明瞭な構造
  • 一次情報源への直接的な参照が欠如する構造
  • 論調が中立性を保とうとする構造
  • 事実の検証可能性が中程度に留まる構造
  • 複数の関係団体の賛否両論を併記する構造

アーキテクチャ（v0.2 本実装）

ENTRY
LINE Bot（Messaging API）／ Chrome Extension（Manifest V3）／ HTTP API ＝ 3 経路から同一 Cloud Run へ集約
RUNTIME
Cloud Run + FastAPI ＝ サーバーレス・min-instances=1・max=10・Secret Manager 連携
AGENT 1 — Watcher
入力テキスト／URL から記事タイトル・要約・主張・感情強度・出典 URL を Gemini で構造抽出
AGENT 2 — Investigator
発信元辞書（信頼ソース 30+）× Gemini で 論調バイアス（0-1） ／ 事実整合性（0-1） ／ 対立見解 ／ 要警戒構造を判定
AGENT 3 — Validator
数値統合（重み付き加算）＋ Gemini で自然言語の構造観察レポート生成（人ではなく構造を見る原則）
DATA
Firestore（信頼ソース辞書／警告履歴）＋ Vertex Vector Search（Phase 3 で過去類似主張照合）
OBSERVE
Cloud Logging — 全判定をログ蓄積＋直近 24h Postmortem 自動生成（SRE 流振り返り）
SDK
google-genai 1.x（旧 google-generativeai は 2025-11-30 で deprecated）／ Gemini 2.5-flash / 新形式 auth key (AQ.Ab～) 対応

構造分析の規律

  • すべての観察文は「○○という構造」「○○の傾向」形式で人物・組織を断定しない
  • 政治的・宗教的に微妙な主張には必ず「複数の立場が成り立つ構造」を contrarian_views で提示
  • 「📚 情報の真偽判断は一次情報まで遡ることを推奨します」を全レポートに常設
  • 北極星 = 社長の信条「真偽は一次情報まで辿る」を AI Agent で社会拡張する設計
